### 今日筆記

#### 1. Handbook 整體架構理解

AI x Web3 Handbook 分為四層地圖：
- AI 基礎：LLM、Prompt、Context、RAG、Agent、Frameworks、MCP、Evaluation
- Web3 基礎：Network、Cryptography、Wallet、Smart Contract、Account Abstraction、DeFi、Oracle、Indexing、Security
- AI x Web3 Bridge：Chain-aware Context、Web3 Tool Use、Agent Workflow、Agent Wallet、Machine Payment 等
- 前沿探索：Agentic Commerce、Wallet/Permission、AI Security、Governance、Dev Tooling、Open Track

關鍵洞察：這個領域不是把兩個 buzzword 拼在一起，而是當 AI Agent 需要真正「執行」鏈上動作時，兩個系統的複雜度會相互放大。

#### 2. LLM 基本工作方式

- LLM 是基於上下文進行概率生成：給定文本 → 預測最合理的下一個 token 序列
- 擅長：語言理解、代碼生成、推理
- 不擅長：精確事實記憶、確定性計算、跨會話狀態保持

#### 3. 四個控制層面

| 層面 | 作用 |
|------|------|
| 上下文窗口 | 工作內存，控制模型能看到多少信息 |
| 系統指令 | 設置身份、語氣和邊界 |
| 提示詞 | 傳遞當前任務意圖 |
| 工具調用 | 讓模型從「說話」變為「做事」 |

#### 4. Prompt → Workflow → Agent 的邊界

- Prompt：讓模型回答，決策在人
- Workflow：預定義任務流程，模型是其中一個節點，路徑固定
- Agent：模型自主規劃、動態調用工具、跨輪管理狀態

風險意識：三者失控風險和可調試性完全不同，越接近 Agent，越需要 guardrails 和 human-in-the-loop。

#### 5. 為什麼 AI 輸出必須驗證

1. 事實錯誤：自信編造不存在的信息
2. 引用錯誤：編造論文、鏈接、數據來源
3. 推理漂移：長上下文中邏輯鏈斷裂
4. 執行越權：agent 做了超出授權範圍的操作
5. 工具誤用：調用了錯誤的工具或參數
