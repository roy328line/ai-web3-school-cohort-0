# 個人學習計劃 · AI × Web3 School

- 學員畫像：AI 有基礎 · Web3 有基礎 · 無代碼 · 產品研究 + Hackathon
- - 每日時間：2 小時
  - - 開始日期：2026-05-17
    - - 輸出語言：繁體中文
     
      - ---

      ## 學習路徑總覽

      以「產品理解 → 交叉概念打通 → Hackathon 原型」為主線。
      不以寫代碼為目標，以能清晰定義問題、拆解系統、提出產品方案為核心產出。

      ---

      ## 第一階段：補齊 AI × Web3 共同語言（Week 1–2）

      目標：能用統一語言描述 AI Agent 與鏈上系統的交叉點。

      | 日程 | Handbook 章節 | 學習重點 |
      |------|--------------|----------|
      | Day 1–2 | LLM + Prompt | 模型能做什麼、Prompt 如何定義邊界 |
      | Day 3–4 | Context + RAG | 資訊如何進入模型、來源可信度 |
      | Day 5–6 | Agent + Frameworks | Agent 如何執行多步任務 |
      | Day 7 | 複盤 + 整理筆記 | 整理第一週概念地圖 |
      | Day 8–9 | Network + Wallet | 鏈上環境、錢包即身份 |
      | Day 10–11 | Smart Contract + Account Abstraction | 鏈上規則、Agent 權限表達 |
      | Day 12–13 | DeFi + Oracle | 開放金融協議、鏈外數據風險 |
      | Day 14 | 複盤 + 整理筆記 | 整理第二週概念地圖 |

      ---

      ## 第二階段：進入 AI × Web3 Bridge（Week 3–4）

      目標：能描述 Agent 如何讀鏈、調工具、管權限、完成支付。

      | 日程 | Handbook 章節 | 學習重點 |
      |------|--------------|----------|
      | Day 15–16 | Chain-aware Context + Web3 Tool Use | 鏈上狀態如何進 Agent |
      | Day 17–18 | Agent Workflow + Agent Wallet | 自動化邊界與權限限制 |
      | Day 19–20 | Machine Payment + Settlement & Escrow | 機器支付與結算邏輯 |
      | Day 21–22 | Agent Identity + Agent Trust & Reputation | Agent 身份與可信歷史 |
      | Day 23–24 | AI Security + AI Privacy | 攻擊面與隱私邊界 |
      | Day 25–26 | Verifiable AI + Governance AI | 可驗證輸出與治理協作 |
      | Day 27–28 | 複盤 + 整理系統地圖 | 畫出 AI × Web3 產品地圖草稿 |

      ---

      ## 第三階段：Hackathon 原型（Week 5–6）

      目標：選擇一個前沿探索賽道，定義最小產品原型。

      推薦賽道（按畫像排序）：
      1. **智能體商業（Agentic Commerce）**：Agent 發現服務 → 協商 → 支付 → 留憑證
      2. 2. **錢包與權限（Wallet / Permission）**：Session Key、Policy 產品設計
         3. 3. **治理（Governance）**：DAO 提案摘要、投票輔助工具
            4. 4. **AI 安全（AI Security）**：攻擊面分析 + 權限隔離展示
              
               5. 每日產出形式：
               6. - 產品 PRD 草稿（無代碼）
                  - - 系統架構圖說明（文字描述 + 圖示）
                    - - 用戶故事 / 場景拆解
                      - - Hackathon Demo 展示文稿
